## Что такое область видимости

Область видимости определяет, где в программе доступна та или иная переменная. Не каждая переменная видна отовсюду — это намеренное ограничение, защищающее данные от случайных изменений.

```python
def greet():
    message = "Привет!"   # эта переменная живёт только внутри функции
    print(message)

greet()
print(message)   # NameError: переменная message не существует здесь
```

## Локальные переменные

Переменная, созданная внутри функции, называется **локальной**. Она существует только во время выполнения функции и недоступна снаружи.

```python
def calculate():
    result = 42   # локальная переменная
    print(result)

calculate()      # 42
print(result)    # NameError
```

Каждый вызов функции создаёт **новые** локальные переменные:

```python
def counter():
    x = 0
    x += 1
    print(x)

counter()   # 1
counter()   # 1  — каждый раз начинается заново
counter()   # 1
```

## Глобальные переменные

Переменная, созданная вне всех функций, называется **глобальной**. Она видна везде в файле.

```python
language = "Python"   # глобальная переменная

def show_language():
    print(language)   # читаем глобальную — это работает

show_language()   # Python
print(language)   # Python
```

Функция может **читать** глобальную переменную, но не может **изменить** её без специального указания:

```python
counter = 0

def increment():
    counter += 1   # UnboundLocalError!

increment()
```

Python считает `counter` локальной переменной (раз есть присваивание), но она не инициализирована — отсюда ошибка.

## Ключевое слово global

`global` сообщает Python, что переменная внутри функции — это та самая глобальная переменная.

```python
counter = 0

def increment():
    global counter
    counter += 1

increment()
increment()
increment()
print(counter)   # 3
```

`global` нужен только когда хотим **изменить** глобальную переменную. Для чтения он не нужен.

```python
# Читать — работает без global
total = 100

def show():
    print(total)   # OK

# Изменять — нужен global
def add(value):
    global total
    total += value

add(50)
print(total)   # 150
```

### Несколько глобальных переменных

```python
x = 0
y = 0

def reset():
    global x, y
    x = 0
    y = 0
```

## Правило LEGB

Python ищет переменную в четырёх областях, в строгом порядке:

| Буква | Область | Описание |
|-------|---------|----------|
| **L** | Local | Внутри текущей функции |
| **E** | Enclosing | Во внешней функции (для вложенных функций) |
| **G** | Global | На уровне модуля (файла) |
| **B** | Built-in | Встроенные имена Python (`len`, `print`, ...) |

```python
x = "глобальная"   # G

def outer():
    x = "внешняя"   # E

    def inner():
        x = "локальная"   # L
        print(x)

    inner()    # локальная
    print(x)   # внешняя

outer()
print(x)   # глобальная
```

Если убрать локальное `x` из `inner`:

```python
x = "глобальная"

def outer():
    x = "внешняя"

    def inner():
        print(x)   # берёт из E (внешняя функция)

    inner()   # внешняя

outer()
```

## Вложенные функции и nonlocal

Когда функция вложена в другую и хочет изменить переменную внешней функции, используется `nonlocal`.

```python
def make_counter():
    count = 0               # переменная внешней функции

    def increment():
        nonlocal count      # указываем, что это переменная из E
        count += 1
        return count

    return increment

counter = make_counter()
print(counter())   # 1
print(counter())   # 2
print(counter())   # 3
```

Разница между `global` и `nonlocal`:

| Ключевое слово | Что делает |
|----------------|-----------|
| `global` | Ссылается на переменную уровня модуля (G) |
| `nonlocal` | Ссылается на переменную ближайшей внешней функции (E) |

```python
x = "global"

def outer():
    x = "outer"

    def inner():
        nonlocal x
        x = "changed by inner"

    inner()
    print(x)   # changed by inner

outer()
print(x)   # global (глобальная не тронута)
```

## Лучшие практики

### Минимизируйте использование global

Глобальные переменные усложняют понимание кода — непонятно, кто и когда их изменил. Лучше передавать данные через аргументы и возвращать через `return`.

```python
# Плохо — скрытая зависимость через глобальную переменную
total = 0

def add_to_total(value):
    global total
    total += value

# Хорошо — явная передача и возврат
def add(total, value):
    return total + value

total = 0
total = add(total, 10)
total = add(total, 5)
print(total)   # 15
```

### Константы пишутся в верхнем регистре

Глобальные переменные, которые не меняются, принято именовать в `UPPER_CASE`:

```python
MAX_ATTEMPTS = 3
DEFAULT_LANGUAGE = "ru"
PI = 3.14159
```

Это сигнал для читателя: «это не должно изменяться».

   double = make_multiplier(2)
