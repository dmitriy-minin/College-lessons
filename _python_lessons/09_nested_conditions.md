## Что такое вложенные условия

Вложенные условия — это операторы `if` внутри других операторов `if`. Они позволяют проверять дополнительные условия только тогда, когда внешнее условие уже выполнено.

Когда нужны вложенные условия:
- Проверка нескольких независимых критериев по очереди
- Уточнение условий после прохождения первичной проверки
- Реализация многоступенчатой логики

---

## Синтаксис вложенных условий

```python
if внешнее_условие:
    # код если внешнее True
    if внутреннее_условие:
        # код если оба True
    else:
        # код если внешнее True, внутреннее False
else:
    # код если внешнее False
```

Простой пример:

```python
age = 20
has_ticket = True

if age >= 18:
    print("Возраст подходит")
    if has_ticket:
        print("Проходите, у вас есть билет")
    else:
        print("Купите билет на входе")
else:
    print("Вход только для совершеннолетних")
```

Вывод:
```
Возраст подходит
Проходите, у вас есть билет
```

## Глубина вложенности

Условия можно вкладывать на несколько уровней. Каждый уровень добавляет 4 пробела отступа.

```python
is_logged_in = True
is_admin = False
has_permission = True

if is_logged_in:
    print("Пользователь авторизован")
    if is_admin:
        print("Привет, администратор!")
        print("Все функции доступны")
    else:
        print("Обычный пользователь")
        if has_permission:
            print("Доступ к разделу разрешён")
        else:
            print("Нет прав доступа к разделу")
else:
    print("Необходима авторизация")
```

Вывод:
```
Пользователь авторизован
Обычный пользователь
Доступ к разделу разрешён
```

### Визуализация уровней

```
if is_logged_in:          # уровень 0
    if is_admin:          # уровень 1
        if ...            # уровень 2
            if ...        # уровень 3
```

Каждый уровень — 4 дополнительных пробела.

## Упрощение вложенных условий

Глубокая вложенность затрудняет чтение кода. Часто вложенные условия можно заменить.

### Замена через and

Два вложенных `if` часто заменяются одним с оператором `and`:

```python
# Вложенный вариант (сложнее читать)
if age >= 18:
    if has_ticket:
        print("Вход разрешён")

# Упрощённый вариант (лучше)
if age >= 18 and has_ticket:
    print("Вход разрешён")
```

### Ранний выход (early return / early continue)

Инвертируйте условие и завершите проверку раньше:

```python
# Вложенный вариант
if is_logged_in:
    if age >= 18:
        if has_subscription:
            print("Показываем контент")

# Упрощённый через ранний выход
if not is_logged_in:
    print("Войдите в систему")
elif age < 18:
    print("Контент только для взрослых")
elif not has_subscription:
    print("Оформите подписку")
else:
    print("Показываем контент")
```

Второй вариант линейный — нет вложенности, легче читать.

## Практические примеры

### Категоризация покупателя

```python
age = 25
is_student = True
purchase_total = 1800

print("Расчёт скидки:")

if age < 18:
    if purchase_total >= 1000:
        discount = 0.15
        reason = "детская скидка + сумма покупки"
    else:
        discount = 0.10
        reason = "детская скидка"
elif is_student:
    if purchase_total >= 2000:
        discount = 0.20
        reason = "студенческая скидка + крупная покупка"
    else:
        discount = 0.15
        reason = "студенческая скидка"
elif age >= 60:
    discount = 0.20
    reason = "пенсионная скидка"
else:
    if purchase_total >= 3000:
        discount = 0.10
        reason = "скидка за крупную покупку"
    else:
        discount = 0
        reason = "без скидки"

final = purchase_total * (1 - discount)
print(f"Скидка: {discount:.0%} ({reason})")
print(f"Сумма к оплате: {final:.2f} руб.")
```

Вывод:
```
Расчёт скидки:
Скидка: 15% (студенческая скидка)
Сумма к оплате: 1530.00 руб.
```

### Определение типа треугольника

```python
a = 5
b = 5
c = 5

# Сначала проверяем, является ли треугольник вообще треугольником
if a + b > c and a + c > b and b + c > a:
    print("Треугольник существует")
    if a == b == c:
        print("Тип: равносторонний")
    elif a == b or b == c or a == c:
        print("Тип: равнобедренный")
    else:
        print("Тип: разносторонний")
else:
    print("Треугольник не существует: сумма двух сторон должна быть больше третьей")
```

Вывод для `a=5, b=5, c=5`:
```
Треугольник существует
Тип: равносторонний
```

### Система доступа к файлу

```python
user_role = "editor"
file_owner = "ivan"
current_user = "maria"
file_is_locked = False

print(f"Пользователь: {current_user}")
print(f"Роль: {user_role}")

if file_is_locked:
    print("Файл заблокирован другим пользователем")
else:
    if user_role == "admin":
        print("Доступ: полный (администратор)")
    elif current_user == file_owner:
        print("Доступ: полный (владелец файла)")
    elif user_role == "editor":
        print("Доступ: редактирование")
    elif user_role == "viewer":
        print("Доступ: только чтение")
    else:
        print("Доступ: запрещён")
```

## Лучшие практики

### Ограничивайте глубину вложенности

Код с вложенностью более 3 уровней трудно читать и поддерживать. Если замечаете такую структуру — рефакторьте.

```python
# Плохо: 4 уровня вложенности
if a:
    if b:
        if c:
            if d:
                print("Всё выполнено")

# Лучше: используйте and
if a and b and c and d:
    print("Всё выполнено")
```

### Выносите сложные условия в переменные

```python
# Труднее читать
if age >= 18 and age <= 65 and not is_blacklisted and (has_insurance or has_guarantee):
    approve_loan()

# Лучше: именованные условия
is_eligible_age = 18 <= age <= 65
is_trusted = not is_blacklisted
has_coverage = has_insurance or has_guarantee

if is_eligible_age and is_trusted and has_coverage:
    approve_loan()
```

### Комментируйте сложную логику

```python
# Пользователь получает VIP-статус если:
# - сделал более 10 покупок, или
# - потратил более 50000 руб. за последний год
if purchase_count > 10 or annual_spending > 50000:
    user_status = "VIP"
```
